理解原型对象

构造函数创建新对象实例：构造函数 Person(), 用 new 构建对象实例 person

  • 每个函数都有一个 prototype 属性, 这个属性指向函数的原型对象。
  • 所有原型对象都会自动获得一个 constructor属性，这个属性包含一个指向 prototype 属性所在函数的指针

proto

  1. 当调用构造函数创建一个新实例后，该实例的内部将包含一个指针（内部属性），指向构造函数的原型对象。
  2. __proto__无法访问, 可通过 isPrototypeOf()确定对象之间是否存在这种关系。
  3. Object.getPrototypeOf()方法可以取得一个对象的原型, 即返回实例__proto__的值。

constructor

每个原型对象都有一个 constructor 属性指向关联的构造函数

原型链(蓝色proto)

优先在实例对象中查找，若无，则通过原型链访问保存在原型对象(可读不可更改)。

判断原型和实例之间的关系

  1. 实例 instanceof 原型: instance instanceof Object
  2. isPrototypeOf(): Object.prototype.isPrototypeOf(instance)

方法

  1. 操作对象 . hasOwnProperty (‘属性’)：检测属性存在于实例?or 原型?

  2. in操作符

    • 单独使用： 无论该属性存在于实例中还是原型中，只要通过对象能够访问到，就返回 true。
    • for-in中使用： 不建议使用

  3. Object.keys (操作对象)： 返回一个包含所有可枚举属性的字符串数组

  4. Object.getOwnPropertyNames (操作对象)： 返回所有实例属性，无论它是否可枚举

每创建一个函数，就会同时创建它的 prototype 对象，这个对象也会自动获得 constructor 属性。而使用对象字面量，本质上完全重写了默认的 prototype 对象，因此 constructor 属性也就变成了新对象的 constructor 属性（指向 Object 构造函数），不再指向 Person 函数
